# Plan for `quran-v4.html`: Immersive "Real Tasmee'" Update

**Objective:** Create a new file, `quran-v4.html`, cloning the base functionality of `quran-v3.html`[cite: 1] but completely overhauling the "Tasmee' Pro" (`#tasmeeProPanel`) feature[cite: 1]. The new feature will discard the text-rendering approach and implement an immersive, image-based word-masking system. The actual Mushaf page image will be displayed with opaque HTML overlays hiding the words, which are progressively revealed via voice recognition, leaving the Ayah end-markers visible.

---

## Phase 1: Initialization & Structure
1.  **File Creation:** Clone the exact contents of `quran-v3.html`[cite: 1] into a new file named `quran-v4.html`. Do not alter the original `quran-v3.html`.
2.  **Panel HTML Replacement:** Locate the `<div id="tasmeeProPanel">` block[cite: 1].
    *   Remove the `<div class="tp-content" id="tpContent"></div>` text container[cite: 1].
    *   Replace it with a new image container: `<div class="tp-image-container" id="tpImageContainer"></div>`. This container will clone the main `mushaf-img` and host the absolute-positioned masking `div`s.
3.  **Control Bar HTML Overhaul:** Redesign the `.tp-bottom-bar`[cite: 1] to match the screenshot:
    *   **Left:** A white pill button displaying the error count (e.g., `أخطاء 0`).
    *   **Center-Left:** Navigation buttons (Next/Prev) and the "Peek" (Eye) button.
    *   **Right:** A prominent, large, floating green Stop button (`#tpStopTasmeeBtn`).

---

## Phase 2: Word-Level Coordinate Extraction
The current `quran-v3.html` groups words into whole-line bounds for the highlight feature[cite: 1]. `quran-v4.html` requires exact word-level bounds.
1.  **Update SQLite Queries (Legacy/Colored):** Create a new function `getWordLevelLegacyCoords(pageNumber)`. Query `ayarects` to return an array of *every single word* (`minx, maxx, miny, maxy`), preserving the `soraid` and `ayaid`.
2.  **Update JSON Mapping (Medina 1441/Bordered):** Create a new function `getWordLevelMedinaCoords(pageNumber)`. Parse the `medinaCoordsByAyah`[cite: 1] but do not merge the horizontal segments. Keep each segment as an independent bounding box representing a word or partial word.
3.  **Marker Exclusion Logic:** Implement a heuristic to identify the Ayah end-marker. For example, assume the last bounding box of any given `ayaid` is the verse circle, and tag it with `isMarker: true`.

---

## Phase 3: The Masking Engine (CSS & DOM)
1.  **CSS Setup:** Add styles for `.tasmee-mask`.
    *   `position: absolute;`
    *   `background-color: var(--bg-page);` (or sample the specific variant's background hex).
    *   `border-radius: 4px;`
    *   `transition: opacity 0.3s ease, background-color 0.2s ease;`
    *   `z-index: 10;`
2.  **DOM Generation (`_tpRenderImageMasks`):**
    *   Clone the current `pageImg.src`[cite: 1] into `#tpImageContainer`.
    *   Calculate the scaling factors using the existing `getVariantHighlightCal()`[cite: 1].
    *   Loop through the word-level coordinates. For every item where `isMarker === false`, append a `.tasmee-mask` `div` over the image.
    *   Assign dataset attributes: `data-surah`, `data-ayah`, and `data-word-index`.

---

## Phase 4: Tarteel Engine (Voice) Integration
1.  **Progressive Reveal Logic (`_aiHandleResult` modification):**
    *   When the `_tarteelWorker`[cite: 1] returns a high-confidence match for a specific word or Ayah sequence, select the corresponding `.tasmee-mask` elements.
    *   Set their style to `opacity: 0;` to reveal the Mushaf text underneath.
2.  **Error Handling:**
    *   If a mismatch or silence timeout occurs, momentarily flash the expected mask's background color to a soft red (`rgba(231,76,60,0.4)`), increment the error counter in the UI, and revert the mask back to the opaque background color.
3.  **The "Peek" Function (`tpToggleText`):**
    *   Modify the eye icon functionality[cite: 1]. When held or toggled, change the opacity of all currently active `.tasmee-mask` elements to `0.15`, allowing the user to faintly see the hidden words. When released or toggled off, revert opacity to `1`.

---

## Phase 5: Cleanup & State Management
1.  **Session Start/Stop:** Ensure `startAITasmee()`[cite: 1] initializes the word-level coordinate fetch and renders the masks before showing the panel.
2.  **Responsive Resizing:** Attach a `ResizeObserver` to `#tpImageContainer` to recalculate mask percentages/positions if the user rotates the device, mirroring the existing `syncOverlay()` logic[cite: 1].
