# Task: Fix Responsive Layout and Overlay Issues in Quran Web App

## The Core Rule (Strictly Enforced)
**DO NOT** modify the existing `quran.html` file. 
You must duplicate the code and apply all fixes to a completely new file named **`quran-V2.html`**.

## Context
The application is a responsive Quran reader. The layout works perfectly on mobile phones (small screens) but has significant CSS layout and positioning bugs on tablets and Windows desktop browsers (large screens). 

## Bug Descriptions & Requirements

### Bug 1: Page Cropping on Large Screens
* **Symptom:** On large screens, the main Quran page image/container is vertically cropped and does not display the full page.
* **Expected Behavior:** The entire page should scale proportionally to fit within the viewport height, minus the top and bottom navigation bars. 
* **Technical Guidance:** Check the `mushafContainer` or the main image element (`mushaf-img`). Ensure properties like `max-height: 100vh`, `object-fit: contain`, or flexbox layout constraints are correctly preventing the image from overflowing its container vertically.

### Bug 2: Ayah Highlight Bleed (Large Screens)
* **Symptom:** The semi-transparent overlay rectangles (used to highlight the currently playing/selected Ayah) extend horizontally across the entire screen width, bleeding far outside the boundaries of the actual Quran page (`mushaf-img`).
* **Expected Behavior:** The highlight overlays must be strictly contained within the exact dimensions of the Quran page image, regardless of screen width.
* **Technical Guidance:** * The wrapper containing the highlights must exactly match the dynamic width and height of the `mushaf-img`. 
    * Ensure `mushafContainer` (or a specific wrapper around the image and highlights) has `position: relative;` and its width does not stretch to 100% of the screen on desktop. 
    * The highlights should be absolutely positioned relative to this specific, constrained container, not the viewport or a full-width parent.

### Bug 3: Top Border Cropping (Bordered & Green Variants)
* **Symptom:** In the "Bordered" and "Green" UI variants, the ornamental top border of the Quran page frame is cropped or obscured at the top of the screen.
* **Expected Behavior:** The top border must be fully visible and not hidden behind the top UI header or cut off by the edge of the screen.
* **Technical Guidance:** This is likely a safe-area margin issue. Add appropriate `padding-top` or `margin-top` to the `mushafContainer` specifically for these variants, accounting for `env(safe-area-inset-top)` or the fixed height of the top navigation bar.

## Execution Steps for the AI
1. Read the existing `quran.html` file to understand the current DOM structure and CSS classes.
2. Create `quran-V2.html`.
3. Implement media queries (`@media (min-width: ...)`) to target the large screen issues without breaking the currently working mobile layout.
4. Fix the highlight absolute positioning relative to the actual image dimensions.
5. Add the necessary top offsets for the bordered variants.
6. Return the fully updated code for `quran-V2.html`.
